# Knora Development Workflow (für Copilot)

## Before committing Vue changes

- Run `cd frontend && npx vite build` to catch template errors (missing end tags, etc.)

## Secrets & API Keys

- NEVER hardcode API keys, even as fallback in test files
- Use `process.env.VAR_NAME` and exit with error if not set

## Starting / Restarting services

- Use `make restart` (kills all + starts clean)
- Do NOT stack multiple `bun run ... &` — creates duplicate processes
- Use `make kill` to stop everything safely

## Confirm Dialogs in Vue

- Use `useConfirm` composable + `ConfirmModal` component
- Destructure at top level: `const { show: showConfirm, options: confirmOptions, ask: askConfirm, onConfirm, onCancel } = useConfirm();`
- Call with: `const ok = await askConfirm({ title, message, confirmText });`
- Place `<ConfirmModal>` INSIDE `<main>`, before `</main>`

## Vue Templates

- Check tag balance after every edit (every `<div>` needs a `</div>`)
- `<ConfirmModal>` must be before `</main>`, not after
- "Element is missing end tag" often means a missing `</div>` several lines above the reported line
